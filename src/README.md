# src — code overview

This file gives a quick orientation to the `src/` folder so contributors know where to look and what to change.

## Important files and folders

- components/
  - Reusable UI components live here. Add presentational and shared components here so pages can import them.

- hooks/
  - Place custom React hooks that wrap query calls, shared state, or other utilities here.

- lib/
  - Utility functions, API clients, and shared helpers.

- routes/
  - Page and route modules. Each route file typically exports route definitions or page components consumed by the router.

- routeTree.gen.ts
  - This file appears generated by the routing tooling (TanStack Router plugin). Do not edit it by hand — update your route modules and re-run the route generator if applicable.

- router.tsx
  - Router setup. This is where routes, route providers, and global route-level behavior (guards, loaders) are wired into the app.

- server.ts
  - Lightweight server bootstrap. Inspect this to see any server-side endpoints, dev server behavior, or SSR hooks present in the project.

- start.ts
  - App entry point (client-side and/or server-side bootstrapping). If you want to adjust client hydration or server startup, start here.

- styles.css
  - Global styles and Tailwind imports (if used). Update this file or your Tailwind config to change global theming.

## How to add a new page/route

1. Create a route file under `src/routes/` (example: `src/routes/About.tsx` or `src/routes/about/index.tsx`).
2. Export the route/page component and route definition according to the project's routing convention.
3. If the project uses automatic route generation, run the route generation step to update `routeTree.gen.ts` (consult the router plugin or build scripts in package.json).
4. Start the dev server (`npm run dev`) and navigate to the new route.

## Where to look for data fetching

- Check `lib/` for API client code.
- Look for usage of `@tanstack/react-query` in components or hooks — that is the recommended place to centralize data fetching and caching logic.

## Styling and design system

- Tailwind CSS and utility libraries are installed; configure Tailwind in the project root (tailwind config file) if present.
- Radix UI and other UI helpers are installed for accessible primitives.

## Running tests (if any)

There are no test scripts in package.json by default. If you add tests, add npm scripts such as `test` and document how to run them.
